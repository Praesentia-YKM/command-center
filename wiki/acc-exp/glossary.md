# 지출 도메인 용어 사전

| 용어 | 설명 |
|------|------|
| 품의 (CST) | 지출의 첫 단계. 부서가 "이만큼 쓰겠다"고 요청하는 행위. EXP_CST 테이블 |
| 원인행위 (OACT) | 품의 승인 후, 외부 거래처와의 계약·발주 등 지출의 법적 근거가 되는 행위. EXP_OACT |
| 결의 (RES) | 회계팀이 지출을 회계상 확정짓는 단계. EXP_RES |
| 지급명령 (PYM_CMD) | 자금팀이 실제 자금을 빼내라고 지시하는 단계. EXP_PYM_CMD |
| 회계전표 (SLIP) | 차변·대변으로 분개된 회계 기록 단위. ACC_SLIP(마스터) + ACC_SLIPDTS(상세) |
| 분개유형 (JRNLZ_TY_CD) | 같은 예산과목에 대해 차/대변 회계계정을 달리 잡는 분류 코드. ACC_JMT 매핑 |
| Multi 화면 | 다중과목·다중거래처를 한 트랜잭션에서 처리하는 화면. 102BMulti(품의/OACT) / 120Multi(결의) / 106Multi(지급명령) |
| CST_STAT | 품의 상태코드. 공통코드 A64. 01(등록)→91(상신)→93(승인)/15(반려) |
| OACT_STAT | 원인행위 상태코드. 공통코드 A68. CST_STAT과 쌍으로 전이 |
| RES_STAT | 결의 상태코드. 공통코드 A69. 02(등록)→88(검토완료)→93(승인)/15(반려) |
| CMD_STAT | 지급명령 상태코드. 공통코드 A61. 01(미결재)→91(상신)→93(전결)/15(반려) |
| SLIP_STAT | 전표 상태. M(미승인) / S(승인). 결의 승인 시 M→S |
| APP_STAT | 전표 결재 상태. 01(미결재) / 93(결재완료) |
| PYM_STAT | 지급 상태. M(미지급) / J(지급완료) |
| esaApp | 전자결재 시스템 연동. easElcAppPopup.do 호출 + iframe 콜백 |
| PKG_EA_RECOM | 결재 단계 SP. 사용자가 설정한 상태값(예: 01)을 결재 단계에 따라 91/92/93으로 즉시 덮어쓴다 |
| 52번 명령전표 | 지급명령 결재완료 시 SP 호출로 생성되는 전표 형식 |
| 공급가액 (SUPP) | 부가세 제외 금액. 예산통제는 공급가액 기준 |
| 총금액 (AMT) | 공급가 + 부가세. ACC_SLIP.SLIP_AMT는 총금액 |
