# 세출/세입 과목 체계

> 유형: 정책 | 관련 entity: `bdg-ebdg-sbj`

## 개요

세출(나가는 돈) 과목과 세입(들어오는 돈) 과목은 각각 별도 트리로 관리됩니다.

| 트리 | 테이블 | LVL 코드 | 계층 |
|------|--------|---------|------|
| 세출 | BDG_EBDG_SBJ | H / M / S | 목그룹 / 목 / 세목 |
| 세입 | BDG_IBDG_SBJ | K / H / M / S | 관 / 항 / 목 / 세목 |

## 세출 과목 트리

```
H 목그룹     예: 인건비
  │
  └─ M 목     예: 정규직 인건비
        │
        └─ S 세목  예: 기본급
              ▲
              │
              └─ EXP_CST_SBJ.EXP_SBJ가 참조 (cross-domain)
```

지출은 항상 **세목(S) 레벨**에서 발생합니다. 목그룹/목은 집계용으로만 사용.

## MARK_ORD — 정렬 안정성을 위한 키

과목 트리의 정렬·검색을 위해 10자리 MARK_ORD 컬럼을 따로 운영합니다.

- 총 10자리. LVL별 2자리씩 (목그룹 2 + 목 2 + 세목 2 + 여유 4)
- 01부터 순차 증가
- 트리에서 한 노드가 추가/이동되면 영향 노드의 MARK_ORD를 SP가 일괄 재계산

```
H 목그룹      MARK_ORD = '01' + '00' + '00' + '0000' = "0100000000"
  └─ M 목     MARK_ORD = '01' + '01' + '00' + '0000' = "0101000000"
        └─ S 세목  MARK_ORD = '01' + '01' + '01' + '0000' = "0101010000"
```

DHTMLX 그리드의 트리 정렬이 MARK_ORD로 ORDER BY되므로, 이 값이 깨지면 화면이 뒤죽박죽 됩니다.

## SP 호출

- **세출**: `SP_CVR_MARKORD_UPDATE` — BDG_EBDG_SBJ의 MARK_ORD 재계산
- **세입**: `SP_CVR_MARKORD_UPDATE_IMP` — BDG_IBDG_SBJ의 MARK_ORD 재계산

편성 반영(`reflect`) 마지막 단계에서 호출되며, DB 방언별로 동일 이름의 SP가 생성되어 있습니다 (MSSQL은 `dbo.` 접두사 필수).

## 세입 과목 (4계층)

세입은 한 계층 더 깊습니다.

```
K 관    예: 지방세
  └─ H 항    예: 보통세
        └─ M 목    예: 주민세
              └─ S 세목   예: 개인분
```

세입 발생(고지서 발행, 수납 등)은 별도 도메인에서 다루며, 본 문서 범위 밖입니다.
