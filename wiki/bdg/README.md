# 회계/예산 (bdg)

> 유형: 도메인 | 관련 레포: ALLSP_STD | 패키지: `src/java/com/allsharp/bdg`

## 배경

지출 도메인이 자금을 "쓰는" 쪽이라면, 예산 도메인은 "얼마를 쓸 수 있는지를 잡는" 쪽입니다.
회계연도 단위로 예산차수를 잡고, 정책/단위/세부사업의 3계층 위계와 세출/세입 과목체계를
편성한 뒤, 지출 단계에서 잔액을 검증·차감합니다.

## 풀려는 문제

- **계층의 깊이**: 정책 → 단위 → 세부 사업 위계가 3단이고, 세출과목도 H/M/S 3계층입니다.
  편성·통제 모두 이 트리 구조를 정확히 따라야 합니다.
- **편성 단계**: 요구(Y) → 조정(J) → 확정(H) 단계마다 별도 행이 쌓이며,
  본예산(M)과 추경(S) 차수가 동시 운영됩니다.
- **대량 업로드**: 한 회계연도 편성은 수만 행 규모. 엑셀 업로드 → 스테이징 →
  SP_CVR_MARKORD_UPDATE(SP) 반영 흐름으로 처리합니다.
- **MARK_ORD 정렬키**: 과목 트리를 10자리 코드(LVL별 2자리)로 표현해 그리드 정렬을 안정화합니다.
- **잔액 차감**: 지출 도메인이 부르는 통제 인터페이스. 공급가액 기준으로 동시성 안전하게 차감해야 합니다.

## 사용자

| 사용자 | 어떻게 사용하는가 |
|--------|--------------------|
| **예산 담당자** | 차수 등록, 엑셀 편성 업로드, 요구/조정/확정 단계 진행 |
| **사업 부서** | 자기 사업 예산을 조회·검토, 필요 시 조정 요청 |
| **지출 화면(cross-domain)** | 품의 저장 시점에 잔액 검증 + 차감 호출 |
| **감사·결산** | 차수별 / 사업별 / 과목별 예산-실집행 비교 |

## 성공 기준

- 차수별 편성 대량 업로드(수만 행)가 단일 트랜잭션에서 일관성 있게 반영
- 과목 트리 MARK_ORD가 LVL 변경·재정렬에도 정합성 유지
- 잔액 차감의 동시성 안전 — 같은 BDG_EBDG_CPL에 동시 지출 시 음수 잔액 0건
