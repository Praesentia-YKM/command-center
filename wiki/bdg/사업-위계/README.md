# 사업 위계 3계층

> 유형: 정책 | 관련 entity: `bdg-mng-cls`, `bdg-mng-biz`, `bdg-dtl-biz`

## 개요

예산은 회계연도 + 사업 위계 + 과목 체계의 조합으로 편성됩니다.
사업 위계는 **정책사업 → 단위사업 → 세부사업**의 3계층 트리이며, 지출 도메인이 직접 참조하는 단위는 가장 하위인 세부사업입니다.

## 3계층 구조

```
BDG_MNG_CLS  (정책사업)
   │ PK: (ACC_YY, MNG_CLS_CD)
   │
   └─ BDG_MNG_BIZ  (단위사업)
         │ PK: (ACC_YY, MNG_BIZ_CD)
         │ FK: MNG_CLS_CD
         │
         └─ BDG_DTL_BIZ  (세부사업)
               PK: (ACC_YY, DTL_BIZ_CD)
               FK: MNG_BIZ_CD, MNG_CLS_CD
               ▲
               │
               └─ EXP_CST_SBJ.DTL_BIZ_CD가 직접 참조 (cross-domain)
```

## 왜 3계층인가

회계 ERP는 공공/사학 회계 기준을 따르며, 예산을 "어떤 정책 → 어떤 사업 → 어떤 세부 활동"으로 분해합니다.
3계층 위계가 있으면 결산·감사에서 다음이 가능합니다.

- 정책 단위 집계: 정책사업 X 전체 집행률
- 사업 단위 비교: 같은 단위사업의 회계연도별 추이
- 세부 단위 추적: 특정 세부사업의 미집행 잔액 즉시 파악

## 코드 체계 (예시)

| LVL | 코드 자릿수 (가변, 사이트별 다름) | 비고 |
|-----|---------|------|
| 정책사업 | 3자리 | 회계연도별 변경 가능 |
| 단위사업 | 6자리 (정책 3 + 단위 3) | |
| 세부사업 | 10자리 (단위 6 + 세부 4) | EXP_CST_SBJ에서 참조 |

## 트리 유지의 함정

- 단위사업의 정책 소속 변경(MNG_CLS_CD 변경)이 회계연도 중간에 발생할 수 있음
- 같은 코드라도 회계연도가 다르면 별도 행 (ACC_YY가 PK 1번)
- 차수 변경(본예산 → 추경) 시 사업 위계는 그대로 유지, 편성행만 새로 생성

## 참조 흐름

지출 화면(EXP_CST_SBJ)에서 사업을 입력하면 다음 키로 BDG_DTL_BIZ를 직접 참조합니다.

```
ACC_YY = 2026
DTL_BIZ_CD = '12345678'   ◄─ 사용자가 선택한 세부사업

   ▼
BDG_DTL_BIZ
   └─ MNG_BIZ_CD (자동 결정) → BDG_MNG_BIZ
         └─ MNG_CLS_CD (자동 결정) → BDG_MNG_CLS
```

정책/단위사업은 owns 체인으로 도달하므로 EXP_CST_SBJ에 별도 FK가 필요 없습니다.
